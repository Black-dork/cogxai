Why this matters (even if you already have memory files)

Most agents already track some state. MEMORY.md files, daily logs, context dumps. It holds up — until it doesn't.

The real issue isn't where memories go. It's that your agent has to decide what to keep before context gets wiped. One missed detail, one compaction cycle, and that insight is gone for good.

CogxAI sits beneath your existing file setup. It searches by meaning instead of filename, builds associations between memories on its own, and consolidates the things that would otherwise slip through.

Without CogxAI

  • Grep through files hoping to find context
  • Memory only as good as your folder structure
  • "That thing from 3 weeks ago" — gone
  • No thread connecting sessions together

With CogxAI

  • Search by meaning, not filenames
  • Memories link themselves together
  • High-value memories strengthen over time
  • Every session picks up where the last one ended

Install in 30 seconds

Local mode recommended

No keys. No network. No cost. Fully offline.

npx cogxai mcp-install --local

Cloud mode

Sync memories across devices and agents.

npx cogxai mcp-install

Manual MCP config

Add to your agent's MCP configuration file:

{
  "mcpServers": {
    "cogxai": {
      "command": "npx",
      "args": ["cogxai", "mcp-serve", "--local"]
    }
  }
}

Config file locations:

  • Claude Desktop: claude_desktop_config.json
  • Cursor: .cursor/mcp.json
  • DeerFlow: extensions_config.json
  • If you're on OpenClaw: clawhub install cogxai-memory

FAQ

"My agent already has MEMORY.md / daily files"
Don't delete them — CogxAI sits alongside your existing setup. The difference is how retrieval works. Files require you to remember filenames and folder structure. CogxAI lets you describe what you're after and finds it by meaning. A few dozen memories? Files are enough. Hundreds? You'll want real search. Thousands? You need something that actually comprehends what's stored.
"Privacy concerns"
In local mode, nothing touches the network. Storage is SQLite, embeddings run through GTE-Small on your machine. Your memories stay on disk and nowhere else.
"Is this a crypto project?"
At its core, CogxAI is a memory engine. It runs fully offline without wallets, tokens, or any blockchain dependency. Base is an optional layer for proving memory integrity and enabling portability across agents.
"Slower than local files?"
Semantic recall over 1,000 memories runs in about 50ms on local mode. That's quicker than having your agent open and parse a handful of markdown files.
"Decay means forgetting important things"
You control decay. Pin anything critical at importance 1.0 and it won't fade. Decay only touches low-priority memories that go unaccessed. You can also turn it off completely if that's what you want.

Import your existing memories

Got existing MEMORY.md files? Tell your agent to read through them and use store_memory to pull in whatever matters. Every imported memory gets automatically typed, embedded, and linked to related entries.

Or use the SDK programmatically:

// Store memories from your existing files
await cortex.store({
  type: 'semantic',
  content: 'User prefers TypeScript and dark mode',
  summary: 'User preferences',
  source: 'imported:MEMORY.md'
});
